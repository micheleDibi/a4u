## `app/db/base.py`

**Scopo**: classe base ORM (`DeclarativeBase`) con metadata che usa una
naming convention coerente per indici e constraint, più due mixin riusabili.

### Costanti

#### `NAMING_CONVENTION: dict[str, str]`

Naming convention SQLAlchemy:

```
"ix": "ix_<table>_<column>"
"uq": "uq_<table>_<column>"
"ck": "ck_<table>_<constraint_name>"
"fk": "fk_<table>_<column>_<referred_table>"
"pk": "pk_<table>"
```

Genera nomi prevedibili che Alembic può citare nelle migrazioni
(`down_revision` reversibile).

### Classi

#### `class Base(DeclarativeBase)`

Base class per tutti i modelli. Usa `metadata = MetaData(naming_convention=NAMING_CONVENTION)`.

#### `class TimestampMixin`

Mixin con `created_at` e `updated_at` (timezone-aware). `created_at`
`server_default=now()`, `updated_at` con `server_default=now()` e
`onupdate=now()`.

#### `class UUIDPKMixin`

Mixin con `id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True),
primary_key=True, default=uuid.uuid4)`.

## `app/db/session.py`

**Scopo**: engine async + session factory, costruiti da `Settings`.

### Esporta

- `engine`: `AsyncEngine` creato con
  `create_async_engine(database_url, pool_size, max_overflow,
  pool_pre_ping=True, echo=False, future=True)`.
- `async_session_factory`: `async_sessionmaker(bind=engine,
  expire_on_commit=False, class_=AsyncSession)`.

`expire_on_commit=False` permette di leggere gli oggetti dopo `commit`
senza ulteriore I/O (importante per pattern `dep yield → commit`).

`pool_pre_ping=True` evita connessioni "stale" reciclate dal pool.

## `app/db/seed.py`

**Scopo**: popolare i dati di lookup (ruoli, permessi, role_permissions
default) e creare/promuovere il bootstrap admin. Idempotente: può essere
eseguito più volte senza side effect.

### Costanti

- `log = get_logger("app.seed")`.
- `ROLE_DESCRIPTIONS: dict[str, str]` — descrizioni IT per ogni ruolo.
- `PERMISSION_DESCRIPTIONS: dict[str, str]` — descrizione IT per ogni codice.

### Funzioni

#### `ensure_seed(db: AsyncSession) -> None`

Funzione async pubblica. Chiama in sequenza:

1. `_seed_roles(db)`
2. `_seed_permissions(db)`
3. `_seed_role_permissions(db)`
4. `_seed_avatar_clip_prompts(db)`
5. `_seed_bootstrap_admin(db)`

Logga `seed_complete` al termine.

> Il chiamante deve fare `commit()` (lifespan e tests lo fanno).

#### `_seed_roles(db) -> None` _(privata)_

Inserisce le righe in `organization_roles` solo se mancanti, leggendo da
`ROLE_NAME_IT` e `ROLE_RANK` (definiti in `core/permissions.py`).

#### `_seed_permissions(db) -> None` _(privata)_

Inserisce le righe in `permissions` solo se mancanti, da `ALL_PERMISSION_CODES`.
Setta `description` da `PERMISSION_DESCRIPTIONS` e `scope="organization"`.

#### `_seed_role_permissions(db) -> None` _(privata)_

Per ogni `(role_code, permission_codes)` in `ROLE_DEFAULT_PERMISSIONS`,
inserisce le righe in `role_permissions` se mancanti. Idempotente.

> Non rimuove righe esistenti che non appartengono più ai default: se i
> default cambiano, va fatta una migrazione esplicita.

#### `_seed_avatar_clip_prompts(db) -> None` _(privata)_

Inserisce in `avatar_clip_prompts` 5 prompt EN (movimenti micro-naturali
di docente universitario: `subtle nod`, `eye contact shift`, ecc.) con
`position` 1..5, `is_active=True`. Idempotente: skippa se la tabella ha
già righe (l'admin può poi modificarle dalla pagina di configurazione).

#### `_seed_bootstrap_admin(db) -> None` _(privata)_

1. Legge `BOOTSTRAP_ADMIN_EMAIL` e `BOOTSTRAP_ADMIN_PASSWORD` da
   `Settings`.
2. Se uno dei due è vuoto: log `bootstrap_admin_skipped reason=missing_env`
   e ritorna.
3. Cerca un utente con quell'email:
   - Se esiste **e** non è platform admin → lo promuove
     (`is_platform_admin = True`), log `bootstrap_admin_promoted`.
   - Se esiste già come admin → no-op.
   - Se non esiste → crea un nuovo `User` con `password_hash =
     hash_password(password)`, `is_active=True`,
     `is_platform_admin=True`, `full_name = BOOTSTRAP_ADMIN_FULL_NAME`,
     log `bootstrap_admin_created`.
