# Dipendenze del worker RunPod Serverless XTTS-v2.
# Versioni pinnate per un build deterministico (vedi anche Dockerfile,
# che installa torch==2.4.1 a parte dall'index CUDA).
#
# - coqui-tts SENZA extra [codec]: torchcodec serve solo con torch >= 2.9.
#   Qui torch e' 2.4.x, quindi non serve. Upper bound <0.26 per evitare
#   che una release futura alzi il requisito minimo di torch.
# - transformers >= 4.45: coqui-tts importa `isin_mps_friendly` (4.45+).
# - numpy < 2 OBBLIGATORIO: i wheel di torch 2.4 e numba/librosa (deps di
#   coqui-tts) usano la ABI numpy 1.x. Con numpy 2.x l'import di torch
#   fallisce con "_ARRAY_API not found / compiled using NumPy 1.x".
coqui-tts>=0.24.0,<0.26
transformers>=4.45.0,<5
runpod>=1.6.0
soundfile>=0.12.0
numpy>=1.26.0,<2.0

# Dipendenze G2P/tokenizer per le lingue CJK di XTTS. Sono "extra"
# opzionali di coqui-tts (NON installate di default): senza, un job in
# queste lingue fallisce a runtime con ModuleNotFoundError (es.
# "No module named 'cutlet'" per il giapponese).
# - Giapponese: cutlet (romanizzazione) + unidic-lite (dizionario MeCab
#   usato da fugashi, dipendenza di cutlet).
# - Cinese: jieba (segmentazione parole) + pypinyin.
# - Coreano: hangul_romanize (usato dal tokenizer XTTS coreano).
cutlet>=0.3.0
unidic-lite>=1.0.8
jieba>=0.42.1
pypinyin>=0.51.0
hangul_romanize>=0.1.0
